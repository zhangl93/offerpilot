# 输出模板

## 快速避坑报告

默认控制在用户可以快速读完的长度。

只保留现有输入和已完成调查能够支持的模块：

- 只有公司信息：保留公司、公开反馈和公司核实问题；省略岗位与待遇。
- 只有JD：保留岗位、待遇和岗位核实问题；省略公司风险与公开反馈。
- 公司与JD都有：使用完整结构。
- 某模块虽然适用但没有找到资料：保留该模块并写“公开信息不足”；不要把“未调查”和“未发现风险”混为一谈。
- “下一步必须问”按实际缺口给1至3个问题，不为凑数生成无关问题。

```markdown
## 初步结论

建议：值得继续沟通 / 核实后再决定 / 谨慎推进 / 暂不建议投入时间 / 信息不足

一句话说明最重要的原因。

## 重点发现

- 公司（适用时）：一项最重要的事实、风险或未知项。
- 岗位（适用时）：实际职责、主体或招聘要求中的关键发现。
- 待遇（适用时）：薪资福利中最值得核实的事项。

## 公开反馈（适用时）

概括反复出现的主题，说明实际打开核对的独立内容数量、时间范围和可信度。不要把搜索命中数或平台展示数写成审阅样本数；没有足够样本时明确写“公开样本不足”。

## 下一步必须问

列出1至3个会实质改变判断的问题，优先覆盖主体、岗位性质和总包中的已知缺口。

## 信息边界

查询日期；主体确认状态；已经覆盖、未找到和无法访问的信息；关键来源链接。
```

## 个人匹配报告

```markdown
## 投递建议

优先投递 / 修改后投递 / 谨慎投递 / 暂不建议投递

## 要求与证据

| 岗位要求 | 类型 | 用户证据 | 判断 |
|---|---|---|---|

## 三个主要缺口

按对筛选结果的可能影响排序，区分“缺少经验”和“简历未体现”。

## 投递前行动

只给三项可完成动作，并标出需要用户确认的事实。
```

## 面试复盘

```markdown
## 复盘结论

说明这是明确问题、原因假设还是无法判断。

| 问题 | 分类 | 依据 | 影响 | 下次动作 |
|---|---|---|---|---|

## 优先重练

给出一个可立即重新回答的问题。不要一次生成大量泛化题目。
```
